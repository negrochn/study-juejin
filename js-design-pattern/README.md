# JavaScript 设计模式核⼼原理与应⽤实践

SOLID 设计原则

- 单一功能原则
- 开放封闭原则
- 李氏置换原则
- 接口隔离原则
- 依赖倒置原则

在 JavaScript 设计模式中，主要围绕**单一功能**和**开放封闭**两个原则来展开。

设计模式的核心思想：**封装变化**。

将变与不变分离，确保变化的部分灵活，不变的部分稳定。



23 种设计模式，分为创建型、结构型、行为型。

- 创建型模式封装了创建对象过程中的变化
- 结构型模式封装的是对象之间组合方式的变化，目的在于灵活地表达对象间的配合与依赖关系
- 行为型模式是将对象千变万化的行为进行抽离，确保我们能够更安全、更方便地对行为进行更改
